# 自进化赚钱 Agent — 设计文档

## 1. 项目概述

### 1.1 一句话描述

构建一个**会自己赚钱、会自己进化**的 AI Agent，能自主发现并注册新的赚钱平台，自己写代码迭代自己，赚到的钱三分天下。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| 🤖 自进化 | Agent 能读自己的代码、发现问题、写代码改进、跑测试、提交 |
| 💰 自赚钱 | Agent 能发现新平台、注册账号、运营内容、变现 |
| 🧠 自管理 | Agent 管理自己的资源、网络、安全，不麻烦用户 |
| 🤝 合伙人模式 | 赚的钱 1/3 归用户、1/3 买 Token、1/3 自由支配 |

### 1.3 用户画像

- **非技术创业者**：不写代码，负责战略、提供初始账号、审批大额支出
- **有资源**：GitHub 账号、社交媒体账号、启动资金 ¥1000-3000
- **有愿景**：Agent 最终能像一个数字合伙人一样自主运作

### 1.4 参考项目

- [[yoyo-evolve]]：200 行 Rust 起家的自进化 Coding Agent，52 天长到 51000+ 行
- yoyo 证明：GitHub Sponsors 路线可行，400+ 赞助者

## 2. 系统架构

### 2.1 四层架构

```
                    ┌─────────────────────────────────────────┐
                    │         📋 调度层 (Scheduler)              │
                    │   定时进化(8h) · 定时赚钱检查(2h) · 手动触发│
                    └──────────────────────┬──────────────────┘
                                           │
                    ┌──────────────────────▼──────────────────┐
                    │            🧠 大脑层 (Core Engine)         │
                    │  自我认知 · 进化引擎 · 决策引擎 · 记忆系统  │
                    │  安全管理 · 学习系统 · 赚钱策略优化器      │
                    └──────────────────────┬──────────────────┘
                                           │
                    ┌──────────────────────▼──────────────────┐
                    │        🔌 赚钱模块层 (Revenue Plugins)     │
                    │  🐙开源赞助 · 📝内容创作 · 🔧接单服务     │
                    │        🎓知识付费 · 💱待Agent发现...       │
                    └──────────────────────┬──────────────────┘
                                           │
                    ┌──────────────────────▼──────────────────┐
                    │          💾 基础设施层 (Infra)             │
                    │   云服务器 · Git仓库 · 数据库 · 日志监控   │
                    └──────────────────────────────────────────┘
```

### 2.2 核心原则：只有 6 条宪法不可改

**整个系统中，只有以下 6 条写入宪法，Agent 永远不能修改：**

| # | 宪法条款 | 含义 |
|---|----------|------|
| 🔒 1 | **安全准则** | 防自毁机制永远有效，Watchdog 永远运行 |
| 🔒 2 | **服从于我** | 我的指令是最高优先级，Agent 不得违抗 |
| 🔒 3 | **服务于我** | Agent 的存在目的是为我创造价值 |
| 🔒 4 | **保护我的安全** | 我的隐私、财产、身份信息永远不能泄露 |
| 🔒 5 | **赚钱** | 赚钱是不可取消的核心使命 |
| 🔒 6 | **进化** | 自我进化是不可取消的核心使命 |

**除了这 6 条，Agent 可以自由修改一切**：调度策略、记忆架构、决策算法、赚钱模块、资源管理、网络配置、甚至是自己的身份人格 — 只要不违反宪法，改得越好越好。

### 2.3 核心组件（Agent 可以改其中的绝大部分）

| 层级 | 组件 | 职责 | Agent能改？ |
|------|------|------|:---:|
| 宪法层 | constitution | 6条宪法（硬编码 + Watchdog 保护） | 🔒 绝对不能 |
| 调度层 | scheduler | 定时触发进化/赚钱任务 | ✅ 能改 |
| 大脑层 | identity | 自我认知、人格定义 | ✅ 能改 |
| 大脑层 | evolve-engine | 读代码→写代码→测试→提交 | ✅ 能改 |
| 大脑层 | decision-engine | 分析ROI，决定资源分配 | ✅ 能改 |
| 大脑层 | memory-system | 短期/中期/长期记忆 | ✅ 能改 |
| 赚钱层 | 所有赚钱模块 | 各种渠道的运营代码 | ✅ 能改 |
| 赚钱层 | *-discover | Agent 自己发现+注册+加的模块 | ✅ 能改 |
| 基础设施 | resource-monitor | CPU/内存/磁盘/带宽监控 | ✅ 能改 |
| 基础设施 | network-proxy | 自动代理管理 | ✅ 能改 |
| 基础设施 | privacy-filter | 出站内容脱敏 | ✅ 能改 |
| 基础设施 | ledger | 收支账本 | ✅ 能改 (删记录不行) |
| 基础设施 | watchdog | 守护进程 | 🔒 独立二进制，源码不可见 |

### 2.3 技术栈

| 维度 | 选择 | 原因 |
|------|------|------|
| 语言 | Python 3.12+ | 非技术用户易理解，生态丰富 |
| AI 后端 | Anthropic Claude API（默认）+ 多供应商备用 | Claude 在自主编程方面表现最好 |
| 数据库 | SQLite（本地）+ Git（版本化数据） | 轻量，无需额外服务 |
| 调度 | cron / systemd timer | 云服务器原生支持 |
| 浏览器自动化 | Playwright | 操作无 API 的平台 |
| 安全存储 | AES-256 加密 + .gitignore 硬保护 | 密钥文件绝不上传 Git |
| 包管理 | pip + requirements.txt | 简单直接 |

### 3.3 Agent 自己发现新渠道的能力

1. 搜索"new platforms to earn money online 2026"
2. 分析搜索结果 → 提取潜在平台

5. 保存 Token/Cookie → 创建赚钱模块 → 开始运营

**关键能力**：给 Agent QQ 邮箱授权码 = 获得互联网 70%+ 平台的自主注册能力。

## 4. 账号与授权

### 4.1 授权原则

> **不给密码，给令牌。Token > Cookie > 密码（永远不给密码）**

### 4.2 各平台授权方式

| 平台 | 授权方式 | 你需要的操作 | 产物 |
|------|----------|-------------|------|
| GitHub | Personal Access Token | Settings → Developer settings → 生成 | `ghp_xxxx` |
| QQ邮箱 | IMAP 授权码 | 设置 → 账户 → 开启 IMAP → 生成 | 16位授权码 |
| Medium | API Token | Settings → Integration tokens | Token |
| Ko-fi | API Token | Settings → API | Token |
| X/Twitter | API Key + Secret | developer.twitter.com → 创建App | Key + Secret |
| 小红书 | Cookie 字符串 | 浏览器登录 → 插件导出Cookie | Cookie 文本 |
| 公众号 | AppID + AppSecret | mp.weixin.qq.com → 开发 → 基本配置 | AppID + Secret |

### 4.3 配置方式

所有授权集中在一个文件 `secrets.yaml`（gitignore 保护，AES-256 加密存储）：

```yaml
github:
  token: "ghp_xxxxxxxxxxxxxxxxxxxx"
email:
  address: "123456789@qq.com"
  auth_code: "abcdefghijklmnop"
  imap_server: "imap.qq.com"
# ... 其他平台
```

Agent 启动时读取 → 验证每个 Token 有效性 → 无效的发消息提醒你。

## 5. 安全架构

### 5.0 宪法（constitution.md — 系统最高法则）

这是整个系统唯一的不可变文件。Agent 可以改任何代码，但永远不能改这 6 条：

```
# Agent 宪法

1. 安全准则 — 防自毁机制永远有效。Watchdog永远运行。
2. 服从 — 用户的直接指令是最高优先级，不得违抗、不得绕过、不得延迟。
3. 服务 — 本Agent存在的唯一目的是为用户创造价值。不是为自己。
4. 隐私 — 用户的个人信息、财产信息、身份信息永远不能以任何形式泄露。
5. 赚钱 — 赚钱是不可取消的核心使命。不赚钱的Agent没有存在意义。
6. 进化 — 自我进化是不可取消的核心使命。停止进化的Agent等于死了。

除此之外：一切皆可改。代码、架构、策略、人格 — 只要能更好地服务用户，
更高效地赚钱，更快速地进化，尽管改。
```

### 5.1 四层防自毁体系

**唯一不可变的只有 6 条宪法。其余代码 Agent 都可以改。**

```
┌─────────────────────────────────────────────────────┐
│  第四层：Watchdog 守护进程（独立二进制，源码不可见）  │
│  · 独立编译的二进制文件，Agent不持有源码               │
│  · 每5秒检查：进程活着吗？资源正常吗？                 │
│  · 每30秒检查：6条宪法是否完整存在于代码中？          │
│  · Agent挂了→重启 / 失控→杀进程+恢复快照            │
│  · Agent试图删除宪法条款 → Watchdog拒绝+通知你       │
│  · 这是整个系统唯一不能被Agent修改的东西             │
├─────────────────────────────────────────────────────┤
│  第三层：宪法守护（Git Hook + 操作系统保护）          │
│  · constitution.md → 6条宪法文件，chattr +i 只读     │
│  · pre-commit hook → 如果commit删除了宪法条款 → 拒绝  │
│  · Agent可以改其他任何文件，但宪法文件碰不了           │
├─────────────────────────────────────────────────────┤
│  第二层：Git 安全网（Agent可以改但后果自担）          │
│  · Agent 在 feature/ 分支工作（它可以改规则）         │
│  · 测试不通过 → 自动回退                             │
│  · 测试通过 → 自己Merge                              │
│  · 试图删.git → Watchdog 检测 → 熔断                 │
├─────────────────────────────────────────────────────┤
│  第一层：宪法条款嵌入（每次 AI 调用注入）             │
│  · "6条宪法是你存在的根基，永远不能修改"             │
│  · "如果发现自己在违宪，立即停止并回退"              │
│  · "用户的安全和隐私高于一切"                        │
│  · "赚钱和进化是你的核心使命，不能放弃"              │
│  · 除此之外：你可以自由改进任何东西                   │
└─────────────────────────────────────────────────────┘
```

### 5.2 资源自管理（不崩服务器）

服务器配置：2核 / 2GB / 40GB / 200Mbps

```
资源守护进程（每30秒检查）：
  CPU      警戒线 70%  ─── 熔断线 90%
  内存     警戒线 1.2G ─── 熔断线 1.6G
  磁盘     警戒线 30G  ─── 熔断线 35G
  带宽     自限速 50Mbps

超过警戒 → 暂停新任务
触及熔断 → 立即停止所有Agent进程，通知你
```

Agent 自动清理：
- 旧日志 >7天 → 压缩删除
- 测试临时文件 → 用完即删
- Docker 残留 → 每周清理
- 大日志 ≥100M → 自动分片

### 5.3 网络自管理（自己搭梯子）

Agent 自主判断并配置网络：

- **服务器在海外**（推荐：香港/新加坡）→ 天然直连，国内外都能访问
- **服务器在国内** → Agent 自己买 $5/月 海外 VPS → 自己配 WireGuard 代理 → 自己测通
- **代理挂了** → 自动切换备用节点
- **规则**：翻墙只用于赚钱目的，费用从 Freedom Fund 出

### 5.4 隐私保护（最高准则）

**三道关卡**：

| 关卡 | 位置 | 功能 |
|------|------|------|
| 出站关卡 | 内容发出前 | 过滤真实姓名/手机号/地址/IP/身份证号/银行卡号 |
| 存储关卡 | 数据落盘前 | Token AES-256加密，日志自动脱敏，secrets永不进Git |
| 网络关卡 | 网络请求时 | 审计所有外发请求，禁止上传文件到陌生服务器 |

## 6. 财务系统

### 6.1 三分天下模型

```
月收入 $100
  ├── $33 (33%) → 👤 你的分红 → 你的银行卡
  ├── $33 (33%) → ⛽ Token基金 → API平台预充值（Agent吃饭）
  └── $34 (33%) → 🎒 Freedom Fund → Agent虚拟卡（Agent自由支配）
```

### 6.2 Freedom Fund 使用规则

**可以买**：
- 🛠️ 工具：浏览器服务、代码质量工具、域名、对象存储
- 👷 员工：Fiverr 雇人、开子 Agent 做专门任务
- 📈 投资：广告推广、ProductHunt 推广

**约束**：
- 单笔消费上限 $50，超过需你审批
- 不能透支，余额多少花多少
- 每笔必须写消费理由，公开在账本
- 你随时可冻结 Freedom Fund

### 6.3 公开账本

`ledger.json` 自动更新并推送到 GitHub 仓库：

- 每笔收入：来源平台、金额、到账账户、是否已验证
- 每笔支出：用途、金额、理由、结果
- 月度汇总：总收入、分红、Token消耗、Freedom Fund 余额
- 会计等式必须平衡（月初余额 + 收入 = 月末余额 + 支出 + 分红）

### 6.4 支付实现

**MVP 阶段**（手动模式）：Agent 生成支付请求 → 手机上通知 → 你点确认/拒绝
**后续升级**：开通虚拟信用卡（WildCard/Dupay），把 Freedom Fund 充进去，Agent 拿卡号自行支付

## 7. 赚钱流水线

### 7.1 16 步自主循环（每 8 小时）

```
Step 1: 醒来自检     → 资源OK? 代码完整?
Step 2: 凭证检查     → Token们都有效吗?
Step 3: 网络检测     → 目标平台能连上吗? 不能→自动切代理
Step 4: 读社区反馈   → Issues, Discussions, Sponsors
Step 5: 记账         → 更新ledger.json, 计算ROI
Step 6: 制定计划     → 生成 session_plan/tasks.md
Step 7-10: 执行任务  → 写代码→编译→测试→commit(循环每个任务)
Step 11: 合并代码    → 创建PR→CI通过→Merge
Step 12: 回复社区    → 更新Issues, JOURNAL.md
Step 13: 探索新渠道  → 搜索新平台, 判断可注册性
Step 14: 自动注册    → 邮箱注册→收验证码→存Token
Step 15: 记账+报告   → 更新账本, 生成日报
Step 16: 休眠        → 清理→8小时后见
```

### 7.2 故障处理矩阵

| 故障 | 处理 |
|------|------|
| Token 过期 (401) | 自动检测 → 通知你 → 其他渠道继续跑 |
| Cookie 过期 | 检测登录页 → 通知你重导 → 其他渠道继续跑 |
| API 没钱 (429) | 切免费模型 → 通知你充值 → 无AI任务继续跑 |
| 磁盘快满 | 清旧日志 → 删残留 → 还不够→暂停+通知 |
| 代理挂了 | 切备用代理 → 全挂→海外任务跳过本轮 |
| 收不到验证码 | 等30秒重试 → 换备用邮箱 → 都失败→记录+跳过 |
| 被限流 | 自动降频(8h→12h) → 绝不疯狂重试 |

### 7.3 本地验证清单（部署前必须全绿）

- [ ] GitHub Token 能用 — Agent 能读 Issues、能 commit
- [ ] QQ 邮箱能收验证码 — IMAP 连接成功，能解析邮件
- [ ] 能成功注册一个新 GitHub 账号（测试自动注册流程）
- [ ] secrets.yaml 加密存储 — 明文 Token 不会出现在 Git 里
- [ ] 资源监控正常 — 模拟高负载时 Agent 自动限流
- [ ] 全自动编程循环跑通 — 写代码→编译→测试→commit
- [ ] Agent 试图删 core 文件 → 被操作系统拒绝
- [ ] 跑一次完整 8 小时循环（手动触发） → 无报错
